# Plan: Animated Custom SVG Logo

## Goal
Replace the current simple sun/cloud `WeatherIcon` logo used in the header brand mark with a custom animated SVG that better fits the CPWeather dashboard and the new dynamic background.

## Current State
- Header logo is rendered in `src/components/dashboard/DashboardHeader.tsx` as `<span className="brand-mark shrink-0 text-white"><WeatherIcon /></span>`.
- `WeatherIcon` is defined in `src/components/dashboard/icons.tsx` and currently contains a simple sun circle plus cloud path.
- `.brand-mark` styling is in `src/index.css` and provides the rounded gradient tile, sizing, and shadow.
- Existing weather animation CSS targets `.hero-weather-icon .weather-icon-sun` and `.hero-weather-icon .weather-icon-cloud`, so header logo animation should use distinct class names to avoid accidental style coupling.
- Reduced motion handling already exists in `src/index.css` and should be extended for the new logo animations.

## Implementation Steps
1. Update `WeatherIcon` in `src/components/dashboard/icons.tsx` with a custom SVG composition suitable for a weather dashboard logo.
2. Keep the same component API and default sizing behavior (`className`, `h-7 w-7`) so existing usages continue to work.
3. Use unique CSS classes for the new SVG layers, such as `brand-weather-logo`, `brand-logo-orbit`, `brand-logo-cloud`, `brand-logo-rain`, `brand-logo-spark`, and `brand-logo-pulse`.
4. Design the SVG as a compact animated weather mark: glowing atmosphere/orbit, cloud mass, rain/signal strokes, and small spark/sun elements.
5. Add animation rules in `src/index.css` near `.brand-mark` or the existing icon animation section.
6. Scope animations under `.brand-mark` or `.brand-weather-logo` so they do not affect hero weather icons or other icons.
7. Add `@keyframes` for subtle float, pulse, shimmer, and dash movement.
8. Extend the existing `@media (prefers-reduced-motion: reduce)` block to disable the new logo animations.
9. Preserve accessibility: keep `aria-hidden="true"` because the logo is decorative beside visible brand text.
10. Run `pnpm lint` and `pnpm build` after implementation.

## Validation
- Confirm the header logo renders correctly on desktop and mobile.
- Confirm no CSS conflicts with `hero-weather-icon` animations.
- Confirm reduced-motion users do not receive continuous SVG animation.
- Confirm `pnpm lint` passes.
- Confirm `pnpm build` passes.
